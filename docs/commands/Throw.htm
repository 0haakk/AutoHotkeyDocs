Throw

Signals the occurrence of an error. This signal can be caught by a try-catch statement.

Throw Value

Parameters

Value

A value to throw; typically an Error object. For example:

throw ValueError("Parameter #1 invalid", -1, theBadParam)

Since this parameter is an expression, all of the following are also valid examples:

throw 3
throw "literal string"
throw MyVar
throw i + 1
throw { what: "Custom error", file: A_LineFile, line: A_LineNumber } ; Throws an object

If omitted, an Error is thrown with a default message.

Remarks

The space or tab after throw is optional if the expression is enclosed in parentheses, as in throw(Error()).

If Try/Catch is in use, Value is assigned to Catch's OutputVar.

If Try is not in use (or is used with Finally but without Catch), the following will occur:

  1. Any active OnError callbacks are called. Each callback may inspect Value and either suppress or allow further callbacks and default handling.
  2. By default, an error message is displayed based on what was thrown. If Value is an Object and owns a value property named Message, its value is used as the message. If Value is a non-numeric string, it is used as the message. In any other case, a default message is used. If Value is numeric, it is shown below the default message.
